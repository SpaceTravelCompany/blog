버전 관리 시스템 Git과 협업 플랫폼 GitHub를 효과적으로 사용하는 방법을 단계별로 설명합니다.

## Git 기본 명령어

### 저장소 관리
```bash
git init          # 새 저장소 초기화
git clone <url>   # 원격 저장소 복제
```

### 변경사항 관리
```bash
git add .         # 모든 변경사항 스테이징
git commit -m "메시지"  # 커밋
git push          # 원격에 푸시
```

### 브랜치 작업
```bash
git branch feature  # 새 브랜치 생성
git checkout feature  # 브랜치 전환
git merge feature   # 브랜치 병합
```

## GitHub 활용 팁

### Pull Request 작성
- 명확한 제목과 설명 작성
- 관련 이슈 링크
- 리뷰어 지정

### Issue 관리
- 라벨을 활용한 분류
- 마일스톤으로 진행 상황 추적
- 템플릿 활용

Git과 GitHub를 능숙하게 다루면 팀 협업이 훨씬 수월해집니다.
