## 1. 패키지와 import

```se
package AA.BB.CC
public package AA.BB.CC

import AA.BB.CC
import engine:AA.BB.CC
import core:AA.BB.CC Alias
```

- `package`는 파일 최상단에 한 번만 올 수 있다.
- `package`에는 `public`, `private`, `internal` 접근 prefix를 붙일 수 있다.
- `import`는 `root:패키지.이름` 형태의 import root와 마지막 alias를 선택적으로 받을 수 있다.
- 패키지 이름은 `.`으로 이어진 식별자 목록이다.

## 2. 선언

### 함수

```se
fn name(a: int, b: int = 0) int {
}

public #inline fn fast() {
}

context(GameCtx) fn update() {
}

context(GameCtx) {
    fn render() {
    }
}

#external fn c_sum(a: int, b: int) int:

#unsafe fn rawWork(ptr: *int) {
}
```

- 함수 prefix: `public`, `private`, `internal`, `#external`, `static`, `#builtin`, `#inline`, `#cold`, `#requireResult`, `#unsafe`, `context(...)`
- 호출 규약 prefix: `#cdecl`, `#stdcall`, `#fastcall`
- 함수 parameter는 `name: Type`, `name = value`, `name: Type = value` 형태를 받는다.
- 반환 타입은 없거나 하나 이상 붙일 수 있다.
- 본문 없는 함수 선언은 끝에 `:`를 붙이며, `#external` 또는 `#builtin` 함수에서만 허용된다.
- `#external`과 `#builtin` 함수는 본문을 가질 수 없다.
- `#external` 함수는 기본적으로 `#unsafe` 함수이며, `#unsafe`와 `#external`은 같이 붙일 수 없다.
- `context(GameCtx)` 함수 안에서 다른 `GameCtx` 컨텍스트 함수를 호출하면 컨텍스트는 암시적으로 전파된다.
- 컨텍스트 속성은 반드시 `GameCtx.delta`처럼 `컨텍스트타입.속성` 형태로 접근한다.
- 전파는 호출할 때 컨텍스트 값을 넘겨주는 규칙이지, 호출된 함수에서 바꾼 값이 호출자에게 되돌아오는 규칙은 아니다.

```se
struct GameCtx {
    delta: f64
}

context(GameCtx) {
    fn update() {
        GameCtx.delta = 10

        context(GameCtx) fn inner() {
            GameCtx.delta = 16
        }

        inner()
        // 여기서 GameCtx.delta는 여전히 10
    }
}
```

위 예시에서 `inner()` 안의 `GameCtx.delta = 16`은 `inner`가 받은 컨텍스트 값만 바꾼다.
`inner()`가 끝나고 `update()`로 돌아오면 `update()`의 `GameCtx.delta`는 그대로 `10`이다.

### 변수

```se
x := 10
x: int = 10
x: int
def value := 10
threadLocal cache: Cache
#align(16) value: i32 = 0
```

- 변수 prefix: `def`, `public`, `private`, `internal`, `#external`, `threadLocal`, `static`, `#align(n)`
- 전역 변수는 반드시 `def` 또는 `threadLocal`이어야 한다.
- `def`와 `threadLocal`은 같은 변수에 같이 붙일 수 없다.
- `def` 는 컴파일 타임 상수 또는 매크로 선언 키워드 이다.

### aggregate

```se
struct Entity {
    id: u64,
    public name: string,

    fn reset() {
    }

    #finalize {
        cleanup()
    }
}

bitStruct Flags {
    enabled: bool
}

prefab Player {
    transform: Transform
}

union Value {
    i: int,
    f: f32
}

rawUnion RawValue {
    i: int,
    f: f32
}

enum Kind {
    A,
    B = 2
}
```

- aggregate 종류: `struct`, `bitStruct`, `prefab`, `union`, `rawUnion`, `enum`
- `struct`, `bitStruct`, `prefab`은 필드, 멤버 함수, 중첩 aggregate, 매크로 호출 형태의 표현식 문장을 받을 수 있다.
- `struct`, `prefab`, `bitStruct`는 `#finalize { ... }`를 받을 수 있다.
- `union`, `rawUnion`, `enum`은 필드/멤버 목록만 받는다.
- aggregate prefix: `public`, `private`, `internal`, `nocopy`, `#align(n)` (internal 은 같은 패키지 묶음)
- 익명 aggregate 타입도 지원한다.
- bitStruct 는 기본적으로는 #align(1) struct 지만 bool 이 1 비트로 취급된다.

```se
value: struct {
    i: int
} = .{ 1 }

field: #align(16) nocopy struct {
    i: int
}
```

### 연산자 오버로딩 함수

`struct`, `bitStruct`, `prefab` 안에서는 연산자 동작을 재정의 할 수있다.

```se
struct Vec2 {
    x: f32,
    y: f32,

    +(other: Vec2) Vec2 {
    }

    [](index: int) f32 {
    }

    ()(scale: f32) Vec2 {
    }
}
```

지원되는 이름: `+`, `-`, `*`, `/`, `%`, `!`, `~`, `==`, `!=`, `<`, `<=`, `>`, `>=`, `+=`, `-=`, `*=`, `/=`, `%=`, `&`, `|`, `^`, `<<`, `>>`, `[]`, `()`

### 테스트

```se
test {
}

test "label" {
}
```

### 매크로

```se
def getter(name, T) {
    quote {
        $name
    }
}
```

- 매크로 선언 키워드는 `def`다.
- `quote { ... }` 안에서 `$name` 형태의 치환 파라미터를 사용할 수 있다.
- quote {} 블럭 안의 내용들이 하드 코딩으로 추가되는 개념이다. 그래서 컴파일타임 조건문, 반복문과도 조합할수 있다.

### 복합 타입

```se

[10]T            // 고정 배열
```

unsafe를 제외하면, 동일 대상에 대해

- `&T`는 동시에 여러 읽기 접근을 허용한다.
- `&mut T`는 동시에 하나의 읽기 쓰기 접근만 허용한다.

다만 언어 디자인 방향은 내부 시스템이나 c 라이브러리 연동 정도를 제외하고 포인터를 거의 안쓰는 방향으로 갈 계획이다.

## 4. 문장

### 블록 / 제어 흐름

```se
{
    //빈 블록
}

if cond {
} else if other {
} else {
}

#cold if cond {
}

for init := 0, init < 10, init += 1 {
}

for cond {//while 과 동작 동일
}

for i in 0:10 {
}

switch value {
    1: {
    },
    else: {
    }
}
```

- 현재 구현된 `for`는 C-style header, while-style header, range-style header를 받는다.
- `break`, `continue`는 loop body 안에서만 허용된다.

### 기타 문장

```se
return
return value
return a, b
defer cleanup()
errdefer cleanup()
break
continue
unreachable
target = value
a, b = c, d
#unsafe {
    cleanup()
}
```

- `#unsafe { ... }` 블록은 함수 본문과 `#finalize` 본문 안에서만 허용된다.
- 함수 본문 안이면 `if`, `else`, `for`, `switch`, 중첩 블록 안에서도 `#unsafe { ... }`를 사용할 수 있다.

## 8. 할당 규칙

GC를 쓰는 객체는 `new`, 필요하면 `delete`를 사용한다.
현재 GC를 강제로 한 번 발동하려면 `#resetGC()`를 사용한다.

C 라이브러리와 주고받는 등 수동 해제 관리가 필요한 메모리는 `#alloc`, `#free` 빌트인을 사용한다.
이 메모리는 `#resetGC()`로 해제되는 대상이 아니다. 당연히 c malloc 함수 그 할당이 아니다!

## 9. 예시

```se
package app

import core:math VecMath

struct GameCtx {
    dt: f64
}

context(GameCtx)
fn update() {
    task0 :task = #runTask(worker)
    result := #join(task0)// 반환값이 없거나 안받아도 되면 안받는다.

    ptr := #alloc(i32, 4)
    errdefer #free(ptr)
    defer #free(ptr)

    if result > 0 {
        render()
    } else {
        unreachable
    }
}

fn worker() int {
    return 1
}
```
