title: RESTful API 설계 원칙

확장 가능하고 유지보수가 쉬운 RESTful API를 설계하기 위한 베스트 프랙티스와 원칙을 알아봅니다.

## REST 기본 원칙

### 1. 리소스 중심 설계
URL은 동사가 아닌 명사를 사용하세요.

```
좋음: GET /users/123
나쁨: GET /getUser?id=123
```

### 2. HTTP 메서드 활용
- `GET`: 조회
- `POST`: 생성
- `PUT/PATCH`: 수정
- `DELETE`: 삭제

### 3. 적절한 상태 코드
- `200`: 성공
- `201`: 생성됨
- `400`: 잘못된 요청
- `404`: 찾을 수 없음
- `500`: 서버 오류

## 추가 권장 사항

### 버전 관리
```
/api/v1/users
/api/v2/users
```

### 페이지네이션
```
/users?page=2&limit=20
```

### 필터링과 정렬
```
/users?status=active&sort=created_at
```

## 문서화

Swagger/OpenAPI를 사용하여 API를 문서화하고, 예제를 제공하세요.

잘 설계된 API는 개발자 경험을 크게 향상시킵니다.
