## 개요

Gmail에서 중요 메일이 오면 Discord로 바로 알림을 보내주는 서비스다.

> 전체 코드: [github.com/SpaceTravelCompany/gmail-discord-no](https://github.com/SpaceTravelCompany/gmail-discord-no) Python으로 작성했고, Gmail API + Discord Webhook 조합으로 동작한다.

```
gmail-discord-no/
├── config.py            # 설정 (.env → 환경변수)
├── gmail_client.py      # Gmail API 래퍼
├── discord_notifier.py  # Discord 웹훅 전송
├── main.py              # 메인 모니터링 루프
├── start.py / stop.py / status.py  # 백그라운드 실행 관리
└── service_wrapper.py   # Windows 서비스 등록
```

---

## 구조

### 1. Gmail 클라이언트 (`gmail_client.py`)

Google OAuth2 인증을 처리하고 Gmail API로 메일을 가져온다. `google-auth-oauthlib`로 로컬 서버를 띄워서 토큰을 발급받고, 이후에는 `token.json`으로 자동 갱신한다.

핵심은 메일 쿼리:

```python
query = "is:important"
if after_timestamp:
    query += f" after:{after_timestamp}"
```

`is:important` 레이블로 Gmail의 중요 메일 분류를 활용한다. 받아온 메일은 ID, 제목, 발신자, 날짜, 본문 스니펫, 중요 여부를 파싱해서 반환한다.

### 2. Discord 알림 (`discord_notifier.py`)

웹훅 URL로 임베드 형태의 메시지를 POST한다.

```
📬 **새로운 중요 메일이 도착했습니다!**
├── 📧 제목
├── 보낸 사람 (이름 + 이메일)
├── 날짜
├── 미리보기 (200자)
└── Gmail 링크
```

200자 초과 스니펫은 자르고, 발신자에 `< >`가 있을 경우 이름과 이메일을 분리해서 보기 좋게 표시한다.

### 3. 메인 모니터 (`main.py`)

60초마다 Gmail을 폴링해서 새 메일을 확인한다.

```
시작 → 기존 메일 100개를 processed_ids에 등록 (무시) → 루프:
  1. Gmail API로 is:important 메일 조회
  2. processed_ids에 없는 메일만 필터
  3. Discord 웹훅 전송
  4. processed_ids에 추가
```

`SIGINT`/`SIGTERM` 핸들러를 등록해서 종료 시 Discord로 종료 메시지를 보내고 깔끔하게 빠진다.

### 4. 백그라운드 실행 (`start.py`)

`pythonw.exe`로 메인 프로세스를 떼고 PID 파일로 상태를 관리한다. `python start.py` / `python stop.py` / `python status.py`로 제어 가능.

Windows 서비스로도 등록 가능한데 (`service_wrapper.py`), `pywin32`로 `win32serviceutil.ServiceFramework`를 상속받아서 Windows 서비스처럼 동작한다.

## 설정

```env
DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/...
GOOGLE_CREDENTIALS_FILE=credentials.json
POLL_INTERVAL=60
IMPORTANT_ONLY=true
```

- `IMPORTANT_ONLY=true`면 Gmail이 중요하다고 판단한 메일만 알림이 간다.
- `POLL_INTERVAL`을 너무 짧게 하면 Gmail API 할당량을 금방 쓸 수 있으니 60초 정도면 적당하다.

## 만들게 된 계기

평소에 회사/개인 메일을 잘 안 보게 된다. 중요한 메일이 있어도 확인을 늦게 하는 경우가 많아서, 자주 보는 Discord로 알림을 빼면 놓치는 일이 줄겠다 싶었다.

처음에는 IFTTT나 Zapier 같은 서비스를 쓸까 했지만, Gmail 중요 메일 필터링 + 자유로운 메시지 포맷이 필요해서 직접 만들었다. 생각보다 간단하게 끝나서 만족 ㅋㅋ

## 팁

- Gmail API OAuth 동의 화면에서 **Test user**로 본인 메일을 등록해야 403이 안 난다.
- `credentials.json`과 `token.json`은 절대 git에 올리지 말 것.
- `IMPORTANT_ONLY=false`로 하면 모든 받은 메일에 알림이 가니까, 테스트할 때만 쓰는 게 낫다. 실수하면 디스코드가 도배된다...
