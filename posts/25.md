## 개요

Gmail에서 중요 메일이 오면 Discord로 바로 알림을 보내주는 서비스예요. Python으로 만들었고, Gmail API랑 Discord Webhook을 같이 썼어요.

> 전체 코드: [github.com/SpaceTravelCompany/gmail-discord-no](https://github.com/SpaceTravelCompany/gmail-discord-no)

```
gmail-discord-no/
├── config.py            # 설정 (.env → 환경변수)
├── gmail_client.py      # Gmail API 래퍼
├── discord_notifier.py  # Discord 웹훅 전송
├── main.py              # 메인 모니터링 루프
├── start.py / stop.py / status.py  # 백그라운드 실행 관리
└── service_wrapper.py   # Windows 서비스 등록
```

---

## 구조

### 1. Gmail 클라이언트 (`gmail_client.py`)

Google OAuth2로 인증하고 Gmail API에서 메일을 가져와요. `google-auth-oauthlib` 덕분에 로컬 서버 띄워서 토큰 받고, 그다음부터는 `token.json`으로 자동 갱신되게 했어요.

핵심 쿼리는 이렇게 생겼어요:

```python
query = "is:important"
if after_timestamp:
    query += f" after:{after_timestamp}"
```

Gmail에서 자체적으로 중요 메일을 분류해주는 `is:important` 레이블을 활용했어요. 가져온 메일은 ID, 제목, 발신자, 날짜, 본문 미리보기, 중요 여부를 파싱해서 돌려줘요.

### 2. Discord 알림 (`discord_notifier.py`)

웹훅 URL로 임베드 메시지를 POST하는 방식이에요.

```
📬 **새로운 중요 메일이 도착했습니다!**
├── 📧 제목
├── 보낸 사람 (이름 + 이메일)
├── 날짜
├── 미리보기 (200자)
└── Gmail 링크
```

200자 넘는 스니펫은 자르고, 발신자에 `< >`가 있으면 이름이랑 이메일을 분리해서 보여줘요.

### 3. 메인 모니터 (`main.py`)

60초마다 Gmail을 폴링해서 새 메일이 왔나 확인해요.

```
시작 → 기존 메일 100개를 processed_ids에 등록 (무시) → 루프:
  1. Gmail API로 is:important 메일 조회
  2. processed_ids에 없는 메일만 거름
  3. Discord 웹훅으로 전송
  4. processed_ids에 추가
```

종료할 때는 `SIGINT`/`SIGTERM` 받으면 Discord로 종료 메시지 보내고 깔끔하게 꺼지게 했어요.

### 4. 백그라운드 실행 (`start.py`)

`pythonw.exe`로 메인 프로세스를 떼놓고 PID 파일로 상태 관리해요. `python start.py` / `python stop.py` / `python status.py`로 조종하면 돼요.

Windows 서비스로도 등록할 수 있어요 (`service_wrapper.py`). `pywin32`로 `win32serviceutil.ServiceFramework` 상속받아서 Windows 서비스처럼 돌게 했어요.

## 설정

```env
DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/...
GOOGLE_CREDENTIALS_FILE=credentials.json
POLL_INTERVAL=60
IMPORTANT_ONLY=true
```

- `IMPORTANT_ONLY=true`면 Gmail이 중요하다고 판단한 메일만 알림이 가요.
- `POLL_INTERVAL`을 너무 짧게 하면 Gmail API 할당량을 금방 다 써버리니까 60초 정도가 적당했어요.

## 만들게 된 계기

평소에 회사 메일이나 개인 메일을 잘 안 보게 되더라고요. 중요한 메일이 와도 확인이 늦는 경우가 많아서, 자주 보는 Discord로 알림을 뺐어요.

처음엔 IFTTT나 Zapier 같은 서비스를 쓸까 했는데, Gmail 중요 메일 필터링이랑 메시지 포맷을 맘대로 바꾸고 싶어서 그냥 만들었어요. 생각보다 금방 끝나서 만족했어요 ㅋㅋ

## 팁

- Gmail API OAuth 동의 화면에서 **Test user**로 본인 메일을 등록해야 403이 안 나요.
- `credentials.json`이랑 `token.json`은 절대 git에 올리면 안 돼요.
- `IMPORTANT_ONLY=false`로 하면 모든 메일이 알림 가니까, 테스트할 때만 쓰는 게 나아요. 실수하면 디스코드가 도배돼요...
