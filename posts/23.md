## 개요

`geometry`는 2D 벡터 그래픽을 Odin에서 처리하기 위해 만든 라이브러리입니다. SVG 파싱, 폴리곤 삼각분할, 클리핑, GPU 래스터화까지 다룰 수 있어요.

> 전체 코드: [github.com/SpaceTravelCompany/geometry](https://github.com/SpaceTravelCompany/geometry)

```cmdstack
geometry/
├── geometry.odin ← 메인: Shape Compute + Loop-Blinn
├── shaders/
│   ├── shape.vert ← 베지어 UV 좌표 전달
│   └── shape.frag ← 음함수 평가 + 안티앨리어싱
├── linalg_ex/ ← 베지어 교차, Subdivision
├── triangulation/ ← 폴리곤 삼각분할 (Ear Clipping)
├── clipper/ ← 폴리곤 클리핑
├── rasterizer/ ← CPU 래스터화 (plutovg)
└── svg/ ← SVG 파싱
```

---

## 왜 Loop-Blinn인가

보통 벡터 곡선을 GPU로 그릴 때는 곡선을 잘게 쪼개서(tessellation) 직선 삼각형으로 렌더링합니다. 근데 확대하면 각져 보이고, 너무 촘촘히 자르면 정점이 엄청 늘어나서..

Loop-Blinn은 곡선을 **수식 그대로 픽셀 셰이더에서 평가**합니다. 삼각분할 아티팩트도 없고 해상도에도 독립적이죠.

---

## 어려움

 Loop-Blinn 자체를 사용해서 곡선을 표현하는것 자체는 생각보다 그렇겐 어렵진 않은데 문제는 삼각분할이 관련 수학 지식이 없으면 어려워서.. Clipper2 라이브러리를 사용하려고 했는데 문제는 C++ 코드라 Odin 으로 직접 바인딩이 안되서 결국 수동으로 포팅했습니다.. 그리고 삼각분할만 잘해도 되는게 아니라 다음 그림 처럼 겹치면 나눠줘야 하는 부분이 있어서;; 여러가지로 힘든 프로젝트였습니다.

 ![Handling Overlapping Triangles](images/23/image.png)

 만약에 쉽고 빠른방법이 필요하다면 그대로 Loop-Blinn을 적용하되 스텐실 버퍼를 사용해서 스텐실 버퍼 기록 모드(반전 모드)로 모양 그리기, 그걸 마스크로 해서 픽셀 렌더링 하면 되는데. 삼각분할 과정이 없어서 쉽고 빠르지만 특히 모바일에서 많이 느리더라고요;; 어쩔수 없이 어려운 방법을 사용했습니다. 그래도 계산 방법자체는 빠른게 장점이라 나중에 또다른 좋은 방법을 구현 할 수도 있을것 같습니다.

## 곡선 타입 분류: `GetCubicCurveType`

4개의 베지어 컨트롤 포인트 `(start, ctl0, ctl1, end)`가 주어지면 이 곡선이 어떤 타입인지 분류해야 합니다. 3차 베지어는 3가지 타입으로 나뉩니다.

**Step 1 — cross 계수 계산**

먼저 f64로 변환하고 세 cross 값을 구합니다:

```
cross₁ = end × ctl₁      (2D cross product)
cross₂ = start × end
cross₃ = ctl₀ × start
```

실제 코드:
```odin
cross1 := [3]f64 { end.y - control1.y, control1.x - end.x, end.x * control1.y - end.y * control1.x }
cross2 := [3]f64 { start.y - end.y, end.x - start.x, start.x * end.y - start.y * end.x }
cross3 := [3]f64 { control0.y - start.y, start.x - control0.x, control0.x * start.y - control0.y * start.x }
```

각 cross는 점과 직선 사이의 signed distance 비슷한 값입니다.

**Step 2 — a₁, a₂, a₃ 계산**

```
a₁ = start · cross₁     (= start.x * cross₁.x + start.y * cross₁.y + cross₁.z)
a₂ = ctl₀ · cross₂
a₃ = ctl₁ · cross₃
```

**Step 3 — d₀, d₁, d₂ (inflection polynomial 계수)**

```
d₀ = a₁ - 2·a₂ + 3·a₃
d₁ = -a₂ + 3·a₃
d₂ = 3·a₃
```

**Step 4 — 판별식 D로 타입 결정**

```
D   = 3·d₁² - 4·d₀·d₂
Δ   = d₀² · D          (최종 판별식 = discr(I))
```

| Δ 부호 | 타입 | 특징 |
|--------|------|------|
| `Δ > 0` | **Serpentine** | S자 곡선, 변곡점 2개 |
| `Δ < 0` | **Loop** | 고리 모양, double point 있음 |
| `Δ = 0` | **Cusp** | 첨점 있음 |

엣지 케이스: `d₀ = 0 && d₁ = 0`이면 **Quadratic** (2차로 퇴화), `d₂ = 0`까지 0이면 **Line**.

> `GetCubicCurveType`은 입력이 f32라도 **내부에서 전부 f64로 계산**합니다. 판별식이 `d₀² · D` 구조라 작은 오차에도 부호가 뒤집힐 수 있어서 그래요.

## 텍스처 좌표 계산: F 행렬

타입 분류가 끝났으면 각 컨트롤 포인트에 `[k, l, m]` 텍스처 좌표를 할당해야 합니다. GPU가 이 좌표를 보간해서 픽셀 셰이더에 넘겨주고.

텍스처 좌표는 4×3 행렬 `F`로 표현됩니다. 각 행 `F[i] = [kᵢ, lᵢ, mᵢ]`은 i번째 컨트롤 포인트의 좌표입니다.

### Serpentine

```
t₁ = sqrt(9·d₁² - 12·d₀·d₂)

ls = 3·d₁ - t₁      lt = 6·d₀
ms = 3·d₁ + t₁      mt = 6·d₀
```

- `ls/lt`, `ms/mt`는 각각 inflection point의 parameter 값
- `L = s·lt - t·ls`, `M = s·mt - t·ms`

```
F[0] = [ ls·ms,          ls³,           ms³          ]
F[1] = [ (3·ls·ms - ls·mt - lt·ms)/3,  ls²·(ls-lt),  ms²·(ms-mt)  ]
F[2] = [ (lt·(mt-2·ms) + ls·(3·ms-2·mt))/3,  (lt-ls)²·ls,  (mt-ms)²·ms ]
F[3] = [ (lt-ls)·(mt-ms),  -(lt-ls)³,  -(mt-ms)³ ]
```

방향 보정: `d₀ < 0`이면 `F` 전체에 `-1`을 곱합니다.

### Loop

```
t₁ = sqrt(4·d₀·d₂ - 3·d₁²)

ls = d₁ - t₁      lt = 2·d₀
ms = d₁ + t₁      mt = 2·d₀
```

```
F[0] = [ ls·ms,  ls²·ms,  ls·ms² ]
F[1] = [ (-ls·mt - lt·ms + 3·ls·ms)/3,  -ls·(ls·(mt-3·ms) + 2·lt·ms)/3,  -ms·(ls·(2·mt-3·ms) + lt·ms)/3 ]
F[2] = [ (lt·(mt-2·ms) + ls·(3·ms-2·mt))/3,  (lt-ls)·(ls·(2·mt-3·ms) + lt·ms)/3,  (mt-ms)·(ls·(mt-3·ms) + 2·lt·ms)/3 ]
F[3] = [ (lt-ls)·(mt-ms),  -(lt-ls)²·(mt-ms),  -(lt-ls)·(mt-ms)² ]
```

방향 보정: `(d₀ > 0 && F[1].k < 0) || (d₀ < 0 && F[1].k > 0)`이면 `F`에 `-1`을 곱합니다.

추가로 double point가 `[0, 1]` 구간 안에 있으면 그 지점에서 곡선을 subdivision 해줘야 합니다. 안 하면 렌더링 아티팩트가 생겨서 그래요.

### Cusp

```
ls = d₂
lt = 3·d₁
```

```
F[0] = [ ls,            ls³,            1.0 ]
F[1] = [ ls - lt/3,     ls²·(ls-lt),    1.0 ]
F[2] = [ ls - 2·lt/3,   (ls-lt)²·ls,    1.0 ]
F[3] = [ ls-lt,         (ls-lt)³,       1.0 ]
```

Cusp는 방향 보정이 필요 없습니다.

### Quadratic (2차)

2차 베지어는 훨씬 간단합니다. `[u, v]` 두 좌표만 쓰면 돼요.

3개 컨트롤 포인트 `(start, ctl₀, end)`의 방향(orientation)을 먼저 확인합니다. 반시계 방향이면 텍스처 좌표 `u` 성분 부호를 음수로 뒤집어서 inside/outside 판별이 안 꼬이게 합니다:

```odin
quadSign := f32(1.0)
if GetPolygonOrientation([start, ctl₀, end]) == .CounterClockwise {
    quadSign = -1.0
}
```

방향 보정 후 텍스처 좌표는 항상 이렇게 들어갑니다:
```
F[0] = [0.0, 0.0, quadSign]     // start → u=0, v=0
F[1] = [0.5, 0.0, quadSign]     // ctl₀ → u=0.5, v=0
F[2] = [1.0, 1.0, quadSign]     // end  → u=1, v=1
```

이 좌표들이 표준 2차 곡선 `v = u²`에 대응됩니다. 셰이더에서 `u² - v`를 평가하고 `quadSign`을 곱해서 최종 부호를 결정합니다.

## GPU 셰이더

### 버텍스 셰이더

```glsl
layout(location = 0) in vec2 inPosition;
layout(location = 1) in vec4 inUv;       // xyz=k,l,m 좌표, w=flag
layout(location = 2) in vec4 inColor;

layout(location = 0) out vec3 fragUv;     // k,l,m
layout(location = 2) out flat uint outFlags;  // 1=CUBIC, 2=QUAD
```

w 채널에 `ShapeVertexFlag`를 넣어서 프래그먼트 셰이더가 2차/3차를 구분합니다.

### 프래그먼트 셰이더

**3차 곡선** — `k³ - l·m` 평가:
```glsl
float c = uv.x * uv.x * uv.x - uv.y * uv.z;
```

**2차 곡선** — `u² - v` 평가:
```glsl
float q = uv.z * (uv.x * uv.x - uv.y);
```

값이 0이면 곡선 위, 양수/음수로 inside/outside가 갈립니다.

### 안티앨리어싱

`fwidth()`로 픽셀 간 변화량을 구해서 coverage를 계산합니다:

```glsl
// Fast AA (기본)
float alpha = clamp(0.5 + c / max(fwidth(c), 1e-6), 0.0, 1.0);
```

Analytic AA 모드도 있는데, `dFdx/dFdy`로 그래디언트 벡터를 구해서 더 정확한 distance-to-curve를 계산합니다. 그래도 품질 차이는 거의 없더라고요.

그리고 직선 쪽 AA는 따로 설정해야되는데 그냥 MSAAx4 적용하면됩니다. 단. vulkan 기준으로 sampleShadingEnable 설정은 끄는게 좋아요! SSAA라 성능영향이 크고 그래서 위의 곡선 AA알고리즘을 쓰고 sampleShadingEnable는 비활성화 하는겁니다.

## 전체 파이프라인: `shapesComputePolygon`

```odin
shapesComputePolygon :: proc(poly: Shapes, allocator := context.allocator) -> (res: RawShape, err: ShapeError)
```

### 입력

`Shapes`는 `ShapeNode` 배열입니다. 각 노드는:
- `pts: [][]Vector2f32` — contour 점 배열 (여러 개의 sub-path)
- `isCurves: [][]bool` — 각 점이 커브 컨트롤 포인트인지 anchor인지
- `color` / `strokeColor` / `thickness`

### 처리 순서

1. **곡선 변환** — 점 배열을 `CurveStructFloat` 리스트로 펼칩니다. `isCurves` 플래그를 따라 Line / Quadratic / Unknown(3차) 타입으로 분류해요

2. **Loop 분할** — 3차 곡선 중 `GetCubicCurveType` 결과가 Loop이고 double point가 `[0,1]` 구간에 있으면 `SubdivCurveAt`으로 그 지점에서 분할합니다

3. **겹침 분할** — 서로 다른 contour의 chord가 겹치는 커브 쌍을 찾아서 최대 3단계까지 subdivision 합니다. chord가 겹친 상태로 Loop-Blinn을 돌리면 셰이더가 inside/outside를 잘못 판별할 수 있어서예요

4. **Loop-Blinn 정점 생성** — 각 커브마다 `_ShapesComputeLine` 호출:
   - Quadratic이면 3개 정점 + 삼각형 1개
   - Cubic이면 4개 정점 + convex hull 삼각분할 (내부 점이 없는 삼각형 2~3개로 쪼갬)
   - 각 정점에 `[k,l,m,flag]` 텍스처 좌표 할당

5. **내부 삼각분할** — 커브 컨트롤 포인트 중 chord 바깥에 있는 것들을 polygon boundary에 삽입한 후, `triangulation.TrianguatePolygons`로 ear clipping 삼각분할합니다. 곡선이 없는 내부 영역을 채우는 삼각형들이에요

6. **출력** — `RawShape` (vertices + indices + bounding rect) 반환합니다. 이걸 그대로 GPU 버퍼에 올려서 그리면 돼요

### 현재 상태와 남은 작업

- **SVG 파싱**과 **clipper**는 아직 테스트 중입니다
- Clipper 모듈은 [Clipper2](https://github.com/AngusJohnson/Clipper2)를 Odin으로 포팅한 것입니다. **오프셋(offset)** 과 **불리언(boolean)** 연산이 필요해서 Clipper2를 선택했습니다. — 단순 클리핑뿐 아니라 폴리곤 오프셋(팽창/수축), 합집합/교집합/차집합 같은 연산을 쓸 수 있어야 외곽선(stroke) 렌더링이 가능해집니다.
- Clipper 연동이 완성되면 현재 비활성화된 `shapesComputePolygon` 내 `clipRect` 기반 클리핑과 외곽선 처리도 활성화할 예정입니다. 지금은 채우기(fill) 렌더링까지만 완성된 상태이고 Clipper가 붙으면 외곽선(stroke) + 오프셋 + 불리언까지 처리할 수 있게 됩니다
