## 개요

| 구분 | OpenGL 4.6 | WebGL 2 |
|------|------------|---------|
| 플랫폼 | 네이티브 (Windows, Linux, macOS) | 브라우저 |
| 기반 | OpenGL 4.6 (데스크톱) | OpenGL ES 3.0 |
| GLSL | 4.60 | GLSL ES 3.00 |
| 용도 | 데스크톱 앱, 게임 엔진 | 웹 브라우저 렌더링 |

WebGL 2는 OpenGL 4.x가 아니라 **OpenGL ES 3.0**을 웹에 포팅한 API다. 따라서 4.6에 있는 고급 기능 대부분은 WebGL 2에 없다.

---

## OpenGL 4.6에만 있는 기능 (WebGL 2 없음)

- **Compute Shader** (GPGPU)
- **Shader Storage Buffer Object (SSBO)**
- **Multi-Draw Indirect** (`glMultiDrawArraysIndirect`, `glMultiDrawElementsIndirect`)
- **Texture Storage** (immutable storage, `glTexStorage*`)
- **Stencil Texturing**
- **Texture Query LOD** (`textureQueryLod`)
- **Program Binary** (플랫폼별 바이너리)
- **Enhanced Layouts** (advanced `layout` qualifiers)
- **SPIR-V** (4.6에서 선택적 지원)
- **Robust buffer access** 등 데스크톱 전용 보장
- **Geometry Shader**, **Tessellation** (4.x 대역; ES 3.0/WebGL 2에는 없음)

---

## WebGL 2 (OpenGL ES 3.0)에서 제공하는 기능

- 2D 배열 텍스처, 3D 텍스처
- 정수 텍스처, 정수 픽셀 연산
- **Transform Feedback** (제한적)
- **VAO 필수** (Vertex Array Object)
- **Instanced Rendering**
- **MRT** (Multiple Render Targets, 제한적)
- **Uniform Buffer Object (UBO)**
- **Sampler Object**
- **Renderbuffer** 포맷 확장
- **Occlusion Query** (제한적)
- **FBO** (Framebuffer Object) 표준 지원

---

## GLSL 차이

| 항목 | OpenGL 4.6 | WebGL 2 |
|------|------------|---------|
| 버전 | `#version 460` | `#version 300 es` |
| 정밀도 | 기본 highp, 지정 선택 | `highp` / `mediump` / `lowp` 권장·필수 |
| 입출력 | `in` / `out` + `layout(location=N)` | 동일 (ES 3.0 스타일) |
| 내장 함수 | 4.60 전 수준 | ES 3.00 수준 (일부 생략) |

---

## API·환경 차이

- **리소스 한계**: WebGL 2는 브라우저별로 텍스처 크기, UBO 크기, varying 개수 등 상한이 있고, 4.6 데스크톱보다 보수적.
- **에러 정책**: WebGL은 검증이 엄격하고, undefined behavior를 최소화하려 함.
- **컨텍스트**: 브라우저 탭/페이지 라이프사이클에 묶임. 컨텍스트 손실 시 복구 절차 필요.
- **확장**: `getExtension()`으로 기능별 활성화. OpenGL 4.6처럼 “버전 올리면 전부 포함”이 아님.

---

## 요약

- **OpenGL 4.6**: 데스크톱 전용, Compute, SSBO, Multi-Draw Indirect, Geometry/Tessellation, 고급 텍스처·레이아웃 등 기능 최대.
- **WebGL 2**: 브라우저 전용, **OpenGL ES 3.0** 수준으로 기능 집합이 작고, 보안·호환으로 제한이 더 많음.

4.6 전용 기능(Compute, SSBO, Tessellation, Geometry Shader 등)을 쓰지 않고, ES 3.0 수준(UBO, MRT, Instancing, VAO, 3D 텍스처 등)만 사용하면 WebGL 2로 이식이 가능한 편이다.
